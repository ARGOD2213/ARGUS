Track 2: Alert Acknowledgement + Escalation
============================================

Files added:
  src/main/java/com/mahindra/iot/service/AlertAcknowledgementService.java
  src/main/java/com/mahindra/iot/controller/AlertWorkflowController.java

Files modified:
  src/main/resources/static/machine.html (ACK buttons + escalation timers)
  src/main/resources/static/index.html (unacknowledged counter card)

Endpoints added:
  POST /api/v1/alerts/{alertId}/acknowledge
    Body: { "operatorId": "OP-01", "note": "Checked zone, NH3 at 18ppm" }
    Response: { alertId, status, acknowledgedBy, acknowledgedAt, note }
    
  GET  /api/v1/alerts/unacknowledged
    Response: List of UnacknowledgedAlert { alertId, machineId, sensorType, value, ageMinutes }
    Sorted by ageMinutes descending (oldest first)
    
  POST /api/v1/alerts/{alertId}/escalate
    Body: { "machineId": "COMP-01", "sensorType": "GAS_LEAK", "value": "28.4" }
    Response: { alertId, action, message }
    Sends SNS notification to aws.sns.topic.arn
    
  GET  /api/v1/alerts/{alertId}/ack-status
    Response: { alertId, acknowledged }
    Boolean check for UI refresh state

Escalation behaviour:
  - Timer starts when CRITICAL alert rendered on machine.html
  - Timer updates every 30 seconds
  - At 3 min unacknowledged: timer turns red with pulse animation
  - At 5 min unacknowledged: auto-POST to /escalate endpoint -> SNS notification sent
  - On operator ACK: escalation timer cleared, UI updated inline, button replaced with green confirmation
  - No page reload required

Storage:
  - DynamoDB iot-sensor-events table
  - Acknowledgement records use "ack#" + alertId as partition key
  - TTL set to 90 days for audit trail retention
  - Separate from original alert# records — no overwrite

ADR compliance:
  - ADR-001: No threshold logic in Spring Boot (escalation is operator-initiated, not control)
  - ADR-002: Zero write to OT layer from ack workflow
  - ADR-003: AiAdvisoryWrapper not applicable (no LLM calls in ack feature)
  - ADR-004: DynamoDB for operational metadata only (acknowledgement records)

Dashboard updates:
  - machine.html: Every CRITICAL alert card displays Acknowledge button + live age timer
    * Button onclick triggers prompt for operator note
    * Timer counts from alert timestamp, formatted as "Unacknowledged Xm"
    * At 3 min: timer transitions to red with pulse animation
    * At 5 min: timer shows "ESCALATED after Xm", button disabled
  
  - index.html summary card: Shows CRITICAL count and unacknowledged count
    * If unacknowledged > 0: Red badge "⚠ X unacknowledged"
    * If all acknowledged: Green check "✓ All acknowledged"
    * Fetches /api/v1/alerts/unacknowledged every 30 seconds
    * Non-blocking — dashboard still loads if endpoint unavailable
